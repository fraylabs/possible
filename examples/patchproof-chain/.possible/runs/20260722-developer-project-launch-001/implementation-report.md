# Developer Project Launch implementation report

## Positioning and truth

- `PRODUCT.md` records durable product truth and unresolved market decisions.
- `launch/positioning/product-truth.md` separates verified behavior from hypotheses.
- `launch/claims/claims-register.md` maps every allowed claim to evidence and prohibits unproven promises.

## Remix

- Exactly three candidates are preserved under `launch/direction/previews/`.
- `tests/remix_previews.py` checks the shared truthful copy and `1440 × 900` viewport.
- `launch/direction/directions.md` records pairwise differences across five material dimensions.
- `launch/direction/decision.json` records the evidence-based, agent-selected Continuous Form direction.

## Launch surface and demonstration

- `launch/site/index.html`, `styles.css`, and `app.js` implement the selected direction without an external runtime asset.
- The demonstration imports tested fixtures and `src/lib/receipt.js`; it does not hard-code claim results.
- Desktop/mobile browser evidence and a machine-readable report live under `launch/evidence/`.
- The page intentionally uses `noindex,nofollow` because no public launch was authorized.

## Adoption path

- Root `README.md` provides start, verification, flow, and boundary information.
- `launch/docs/quickstart.md` is the tutorial; `evidence-model.md` is reference/explanation.
- `launch/examples/passing-draft.json` is a synthetic tested import example.
- `launch/assets/provenance.md` accounts for every launch asset class.
- The clean-room report verifies `npm ci` and the integrated suite from an immutable archive.

## External gate

No deployment, publication, push, tag, release, DNS or provider mutation, analytics, outreach, customer-data collection, account creation, or spending was performed. The product is prepared locally, not launched publicly.

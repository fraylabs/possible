This guide covers an external custom-part quote handoff through
[Protolabs](/wiki/protolabs). A host agent may submit a revisioned model and
drawing package for live DFM feedback and quote configuration, but Possible does
not upload files, authorize disclosure, pay, or fabricate a part.

Use this guide only after live process and file checks and explicit approval of
the proprietary upload. Protolabs must remain suitable on current project
evidence and the revisioned model, drawing, and inspection intent must be ready.
[Xometry quote guidance](/wiki/request-xometry-quote) is related reading, not an
automatic provider recommendation.

Review returned DFM feedback before accepting any functional change. Select and confirm the [Inspection plan](/wiki/inspection-plan) requirements during quote review rather than assuming a generic inspection option will satisfy them.

Do not proceed if the approach assumes unconfirmed material, inspection,
tolerance, facility, or geography support. The approval gate exists because the
upload discloses proprietary design files and creates external DFM and quote
state.

One common host-side approach is:
1. Live-check process, file, material, facility, inspection, tolerance, and destination constraints.
2. Present the exact package and external disclosure for user or project-owner approval.
3. Upload only the approved revision and request DFM plus quote without ordering.
4. Return feedback, quote assumptions, inspection options, and revision identity.

A useful host-side record includes the live quote, DFM feedback, inspection
options, disclosed package revision, and approval used. Reading this guide is
not evidence that a quote was requested or reviewed.
